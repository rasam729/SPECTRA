Movement Simulation

1. The "Virtual Joystick" Logic
On your Guest Web App (index.html), you will add four directional buttons (North, South, East, West). Instead of moving a character in a game, these buttons will update the Latitude/Longitude (or X/Y coordinates) in your Firestore database.

The Logic:

Each click adds or subtracts a small value (e.g., 0.0001) from the current position.

The Guest App "listens" to its own position. Whenever the position changes, it triggers the Text-to-Speech or Vibration logic based on how close it is to a Hazard or an Exit.

2. The Code: Movement & Firebase Sync
Paste this into the JavaScript section of your Guest App. This ensures that every "step" the guest takes is instantly visible to the Responder.

JavaScript
// Initial Position
let currentPos = { x: 50, y: 50 }; // Represented as percentages of the floor plan

function moveGuest(direction) {
    const stepSize = 2; // Move by 2% of the map per click

    if (direction === 'up') currentPos.y -= stepSize;
    if (direction === 'down') currentPos.y += stepSize;
    if (direction === 'left') currentPos.x -= stepSize;
    if (direction === 'right') currentPos.x += stepSize;

    // 1. Update the local UI (The Guest sees themselves moving)
    updateGuestMarker(currentPos);

    // 2. Sync to Firebase (The Responder sees the movement)
    db.collection('users').doc(userId).update({
        location: currentPos,
        lastUpdated: firebase.firestore.FieldValue.serverTimestamp()
    });

    // 3. Trigger Intelligence (Check for directions)
    checkProximityToHazards(currentPos);
}
3. Tracking on the Responder Dashboard
On the Responder Dashboard (dashboard.html), you set up a "Snapshot Listener." This tells the dashboard: "Whenever any guest's location changes in Firebase, move their icon on my screen immediately."

The Dashboard Code:

JavaScript
db.collection('users').onSnapshot((snapshot) => {
    snapshot.docChanges().forEach((change) => {
        if (change.type === "modified") {
            const data = change.doc.data();
            const guestId = change.doc.id;
            // Move the icon on the Responder's map
            moveIconOnDashboard(guestId, data.location);
        }
    });
});

4. Simulating the "Automatic" Directions
To make the demo look professional, the directions shouldn't be manual. They should be Reactive.

Define the Exit: Save the coordinate of the "Nearest Exit" in Firebase.

Distance Calculation: In the Guest App, calculate the distance between currentPos and exitPos.

The Trigger: * If the Guest moves closer to the exit: The phone says (TTS) "Continuing on safe path."

If the Guest moves away (closer to a simulated Hazard): The phone Vibrates (Stealth) and says "Turn back. Hazard detected ahead."



Automating “Hostage Detection” via Noise Sensors

You specifically asked about switching to vibration based on silence. Instead of a button, you use the Web Audio API to turn the Guest’s phone into a “Live Noise Sensor.”

The Code for the Guest App (index.html):
JavaScript
Const audioContext = new (window.AudioContext || window.webkitAudioContext)();
Const analyser = audioContext.createAnalyser();

Navigator.mediaDevices.getUserMedia({ audio: true }).then(stream => {
  Const source = audioContext.createMediaStreamSource(stream);
  Source.connect(analyser);
  
  Function checkSilence() {
    Const dataArray = new Uint8Array(analyser.frequencyBinCount);
    Analyser.getByteFrequencyData(dataArray);
    
    // Calculate volume
    Let volume = dataArray.reduce((a, b) => a + b) / dataArray.length;

    If (volume < 5) { // If it’s extremely quiet
      // UPDATE FIREBASE: “Guest is in a quiet zone (possible hostage/stealth)”
      Db.collection(‘users’).doc(userId).update({ stealthMode: true });
      
      // TRIGGER VIBRATION INSTEAD OF VOICE
      Window.navigator.vibrate([500, 200, 500]); 
    }
    requestAnimationFrame(checkSilence);
  }
  checkSilence();
});


Heat Signature Detection

Integrating heat signature detection to find trapped guests is a high-level "Life Safety" feature that makes your project stand out. In a real-world scenario, you would use IoT Thermal Sensors (like the FLIR Lepton) mounted on the ceiling or as part of a "FireBot" scout.

To implement this without a physical thermal camera, you can simulate the data logic in your Firebase/Web setup.

1. Differentiating Human vs. Fire (The Science)
Fire and humans look very different to an infrared sensor. You differentiate them using three key data points:

Feature          Fire Signature                             Human Signature
Temperature      High (200°C to 1000°C+)                    Low/Consistent (30°C to 40°C)
Shape            "Amorphous, flickering, grows upwards"     "Distinct ""Blob"" (head/shoulders) or lying down"
Movement         Rapid flickering/expansion                 "Slow, rhythmic (breathing), or static/trapped"

The "Smart Sensor" Logic (Pseudo-code)JavaScript

// Function to process simulated thermal data
function analyzeThermalData(sensorId, temperature, shapeMatchScore) {
  // 1. Differentiate from Fire: Fire is > 100°C. Humans are ~37°C.
  if (temperature > 35 && temperature < 42) {
    
    // 2. Shape Matching: AI checks if the "blob" looks like a body
    if (shapeMatchScore > 0.8) {
       // 3. Mark as "Likely Trapped Person" in Firebase
       db.collection('trapped_entities').add({
         location: getSensorLocation(sensorId),
         type: 'Thermal Signature (Human)',
         confidence: shapeMatchScore,
         sos_clicked: false // They haven't clicked the button, but we found them!
       });
    }
  }
}


3. How to show this in your Design
In your Responder Dashboard, you should have a specific visual style for these "Automated Detections" so the responder knows the difference between an SOS and a Sensor Hit.

SOS Button Click: A solid Red Icon with a pulsing ring.

Heat Signature Detection: A Purple/Faint Magenta Blob or a "Humanoid Ghost" icon.

The Sidebar: Add a "Detected Entities" list that says: "Possible human signature detected in Room 204. No SOS signal received. Likely unconscious or trapped."4.
